# 定点観測ストレス評価システム (リアルタイム表情分析＆STAI連動機能付き)

本システムは、PC/USBウェブカメラなどの映像からリアルタイムで人物の表情（感情）および顔特徴（まばたき、頭部運動、口・瞼の緊張など）を抽出し、平常時のベースライン状態からの逸脱（zスコア）を用いてストレス評価値を算出するシステムです。

さらに、心理的不安指標である **STAI（状態・特性不安質問紙）** の回答結果と連動させ、機械学習（線形回帰/リッジ回帰）によって各特徴量の重要度（重み）やストレス値の尺度設定（ゲインとベースライン）を個人の実データに基づいて自動チューニングする機能を備えています。

---

## 💡 主な機能と特徴

### 1. リアルタイム表情・複数特徴分析＆ストレス値評価
*   **MediaPipe FaceLandmarker (Tasks API)**: 毎フレーム顔の478点ランドマークおよび顔表情（blendshapes）を高精度かつ軽量に検出。まばたき、眉間のしわ、瞼や口唇の緊張状態をリアルタイムで追跡します。
*   **HSEmotion (EfficientNet系オンプレミスモデル)**: 感情認識モデル（`enet_b2_8`）を統合し、表情から8クラス（怒り、軽蔑、嫌悪、恐怖、喜び、無表情、悲しみ、驚き）の確率分布を時間平滑化（EMA）を適用しながら安定検出します。
*   **マルチモーダル・ストレス算出**: 先行研究に基づき、以下の6つの成分のzスコアからストレス値をリアルタイム評価（0〜100点）します。
    *   **感情感情値 (emotion)**: ネガティブ感情の確率和
    *   **眉間のしわ (brow)**: 眉を寄せる強さ（眉間しわ AU4）
    *   **まばたき率 (blink)**: 直近30秒間の瞬目頻度（回/分）
    *   **頭部運動 (head)**: 頭部姿勢（回転行列）の角速度
    *   **口唇の緊張 (mouth)**: 唇を強く結ぶ強さ（AU23-24）
    *   **瞼の緊張 (eye)**: 目を細める/見開く強さ（瞼緊張 AU7）
*   **歩行・移動フィルタ**: 頭部の並進移動速度を監視し、歩行や大きな体の動きによるノイズを自動でゲート（抑制）することで、「全身の移動」を「頭単体のストレス運動」と誤評価するのを防ぎます。

### 2. 平常状態のパーソナライズ較正 (Calibration)
*   起動直後の7秒間、カメラの正面でリラックスした平常状態を維持する「較正フェーズ」を設けています。
*   このフェーズで6つの特徴量の平均値と標準偏差（ベースライン）を個人ごとに測定し、以降の評価はこの基準値からの「ズレ（zスコア）」を基に行うため、表情の癖や姿勢による個人差を排除した高精度なストレス評価が可能です。

### 3. 顔識別と人物IDトラッキング (Face ID Tracking)
*   **深層顔認識 (DeepFace SFace / InsightFace)**: 幾何ランドマーク情報だけでは区別が難しい「別人」の分離を高精度に行うため、深層学習を用いた顔埋め込み（Embedding）を利用します。
*   **フレームをまたぐ安定性 (Hysteresis)**: 一瞬の向きの違いや誤検知でIDが頻繁に切り替わらないよう、数フレーム連続で別人と判定されて初めてIDを切り替えるヒステリシス制御（switch patience）を導入。
*   **セッション間の固定ID**: 測定結果や学習した平常統計は人物プロファイル（`person_profiles.json`）に永続化され、再起動後も同じ顔に対して同じIDを割り当てて測定を再開できます。

### 4. 対話型STAI質問紙と自動採点機能 (`survey` モード)
*   セッション終了時に、コンソール上で対話的に **STAI (State-Trait Anxiety Inventory: 状態・特性不安質問紙)** に回答できます（20項目または40項目）。
*   逆転項目（質問文の否定表現など）の採点を自動で行い、状態不安（STAI-S）および特性不安（STAI-T）の得点（20〜80点）を算出。セッションの平均特徴zスコアと合わせて `stai_dataset.jsonl` に保存します。

### 5. `tune_stress.py` による自動最適化 (Tuning)
*   蓄積されたSTAIスコアとセッション平均特徴量のデータセットを読み込み、STAI-S得点に最も連動（高い正の相関）するよう、6つのストレス成分の重み・ゲイン・ベースレベルを自動で最適化します。
*   サンプル数が少ない場合は過適合を防ぐために正則化（リッジ回帰）やスケール調整のみのフォールバックを自動で行い、最適化された設定は `stress_config.json` に即座に適用されます。

## 📂 ファイル構成

| ファイル名 | 役割・説明 |
| :--- | :--- |
| **`peroson_detect.py`** | 本システムの本体スクリプト。カメラ映像のキャプチャ、リアルタイム分析、UI描画、ログ保存、STAI質問紙の実行を処理します。 |
| **`tune_stress.py`** | オフラインチューニングスクリプト。データセットから相関を評価し、最適なパラメータを推定して設定ファイルへ適用します。 |
| **`stai_items.json`** | STAI質問紙のタイトル、項目文、逆転項目インデックス、回答選択肢アンカーが格納された外部定義ファイル。 |
| **`stress_config.json`** | 動作モード（`run`/`collect`/`survey`）、各特徴の重み、ゲイン、標準偏差の下限値（std_floors）等を格納した設定ファイル。※起動時に自動生成されます。 |
| **`run.ps1`** | Windows環境用の開発/実行用ショートカットPowerShellスクリプト。 |
| **`Makefile`** | UNIX/Windows（Make環境）用ビルド・実行タスク自動化ファイル。 |
| **`person_profiles.json`** | 登録された人物の顔埋め込みデータや平常状態（較正結果）の統計情報を格納するデータベース。 |
| **`session_history.jsonl`** | 各測定セッションの要約履歴。 |
| **`stai_dataset.jsonl`** | チューニング用データセット（セッション平均特徴量 ＋ 入力されたSTAI得点）。 |

## 🛠️ セットアップ手順

### システム要件
*   Python 3.8 〜 3.12 (推奨)
*   ウェブカメラ (内蔵またはUSB接続)
*   Windows / macOS / Linux

### 必要ライブラリのインストール
ターミナルまたはPowerShellを開き、以下のコマンドを実行して必要なライブラリをインストールしてください。

```bash
pip install opencv-python mediapipe hsemotion-onnx onnxruntime tf-keras matplotlib deepface
```

*   **より高精度な顔認識 (推奨・任意)**
    顔識別の精度をさらに高めたい場合は、`insightface` をインストールすることも可能です（※Windows環境ではコンパイラの導入が必要な場合があります）。
    ```bash
    pip install insightface
    ```

### モデルファイルの準備
必要なモデルファイル（MediaPipe FaceLandmarkerモデルやDeepFace用SFace重みファイルなど）は、**スクリプトの初回起動時に自動でダウンロードおよび配置**されます。手動で準備する必要はありません。

## 🚀 基本的な使い方

### 1. アプリケーションの起動
以下のいずれかの方法で起動します。

*   **Pythonコマンドから直接実行:**
    ```bash
    python peroson_detect.py
    ```
*   **PowerShell経由 (Windows):**
    ```powershell
    .\run.ps1 run
    ```
*   **Makeコマンド経由:**
    ```bash
    make run
    ```

### 2. 較正 (キャリブレーション) フェーズ
1.  起動するとカメラウィンドウが立ち上がり、上部に赤字で **「CALIBRATING (KEEP NEUTRAL)... 7.0s」** のようにカウントダウンが表示されます。
2.  この間は**カメラの正面を向き、リラックスして表情を動かさない無表情の状態**をキープしてください。
3.  7秒が経過すると較正が完了し、**「CALIBRATED (STRESS DETECT MODE)」** に切り替わり本評価が開始されます。

### 3. ストレス分析中のUI表示
画面上に検出された顔を囲む枠と人物ID（例: `ID 1`）、およびリアルタイムのストレス度（0〜100）が表示されます。
また、右側には現在の各成分値（表情、眉間しわ、まばたき、頭部運動、口・瞼の緊張度）とそれらに基づくストレスバーが表示され、ストレス度合いに応じて色（緑＝低、黄＝中、赤＝高）が変化します。

### 4. 測定の終了と保存
画面がアクティブな状態で **`q` キー** を押すか、コンソールで **`Ctrl + C`** を押すと測定が終了します。
終了時に、以下の2つのファイルが自動保存されます。
*   `stress_log.csv`: 時系列での詳細な特徴量とストレス値のログ。
*   `stress_graph.png`: セッション中のストレス値の推移を表したグラフ画像。

## 📈 STAI連動によるパラメータ調整手順

本システムは、初期状態では標準的なパラメータが設定されていますが、実際のユーザーのSTAI回答と連動させて評価式を最適化できます。

### ステップ1: 動作モードの変更
`stress_config.json`（一度起動すると自動生成されます）を開き、`"mode"` の値を `"survey"` に変更します。

```json
{
  "mode": "survey",
  ...
}
```

### ステップ2: データの蓄積（複数セッションの実施）
1.  システムを起動し、ストレスを伴うタスク（PC作業、計算タスクなど）またはリラックスするタスクを数分間行います。
2.  `q` キーでシステムを終了すると、コンソール上で自動的に「STAI-S 状態不安」および「STAI-T 特性不安」の問診（20〜40項目）が開始されます。
3.  画面の指示に従い `1` 〜 `4` の数値で回答を完了させます。
4.  回答が完了すると、自動採点されたSTAI得点と特徴データが `stai_dataset.jsonl` に保存されます。
5.  ※相関の精度を高めるため、**様々なストレス状況下で最低 4〜5 セッション以上**のデータを蓄積してください。

### ステップ3: チューニングスクリプトの実行

#### A. 現在の相関関係の評価
設定ファイルを書き換えずに、蓄積されたデータと現在の評価パラメータの連動度合いを評価します。
```bash
python tune_stress.py --report
```
コンソールに、各成分とSTAI-Sのピアソン相関係数（$r$）、推定値のRMSE/MAEが表示されます。

#### B. パラメータの自動最適化と適用
蓄積されたデータからSTAI-Sへの相関が最大になるように重み・ゲインを再計算し、`stress_config.json` を更新します。（古い設定は `stress_config.json.bak` としてバックアップされます）。
```bash
python tune_stress.py --apply
```
適用後、`stress_config.json` の `"mode"` を `"run"` に戻すことで、次回以降は最適化されたパーソナライズパラメータでストレス評価が行われます。

## ⚡ Coral USB Accelerator (Edge TPU) 対応（任意）

感情分類（HSEmotion）を USB接続の Coral Edge TPU にオフロードできます。Coral未接続/未設定の環境では
自動的に従来どおりCPU(onnxruntime)で動作するため、無効化していれば挙動は一切変わりません。

### 有効化手順
1. **Coral Edge TPU runtime（USBドライバ）** を導入する（[coral.ai/software](https://coral.ai/software/) の
   "Edge TPU runtime" をOS別にインストール）。
2. **推論用ライブラリ**: `pip install tensorflow`（`tensorflow.lite.Interpreter` を使用。Python 3.9以下の
   環境なら軽量な `tflite-runtime`/`pycoral` でも可）。
3. **Edge TPU用モデルの変換**（Linux/WSL上で一度だけ）:
   ```bash
   cd face_detect/coral
   pip install onnx onnx2tf onnxruntime tensorflow numpy opencv-python-headless
   # Coral公式リポジトリから edgetpu_compiler を導入（Linux x86-64専用）
   python convert_emotion_model.py --model-name enet_b2_8 --calib-dir ./calib_faces
   ```
   `--calib-dir` には顔クロップ画像を20〜100枚程度置くことを推奨します（無いとランダム画像で量子化され精度が落ちます）。
   生成された `models/enet_b2_8_edgetpu.tflite` を Windows側の同じ場所（`face_detect/coral/models/`）にコピーします。
4. `stress_config.json` を編集して有効化します。
   ```json
   {
     "coral": {
       "enabled": true,
       "model_path": "coral/models/enet_b2_8_edgetpu.tflite",
       "device": ""
     }
   }
   ```
   `device` は Coral を複数接続する場合のみ `":0"` 等を指定します（通常は空文字でOK）。

### 注意点
* `edgetpu_compiler` は Linux(x86-64) 専用です。変換はWSL/Docker/Linux上で行い、本体(`peroson_detect.py`)は
  Windows上でそのまま変換済み `.tflite` を読み込むだけです。
* Coral公式には既製の「感情分類」Edge TPUモデルが存在しないため、上記スクリプトで自前変換する必要があります。
  int8量子化により、CPU(float32)版に比べて精度がわずかに劣化する場合があります。導入後は `stress_log.csv` を
  CPU版と見比べることを推奨します。
* Coral未接続・モデル未配置・ライブラリ未導入など、いかなる理由でも起動時に自動でCPUにフォールバックし、
  コンソールにその旨が表示されます（エラー終了はしません）。

## 🧹 データのリセット方法
蓄積データや学習した人物のプロファイルをリセットしたい場合は、以下のコマンドを使用します。

*   **一時的なセッション出力（CSV/PNG/pycache）の削除:**
    ```bash
    make clean    # または .\run.ps1 clean
    ```
*   **人物データベース（プロファイルや平常状態）の削除:**
    ```bash
    make clean-profiles    # または .\run.ps1 clean-profiles
    ```
*   **すべての生成データをリセット:**
    ```bash
    make clean-all    # または .\run.ps1 clean-all
    ```
